Use Hexo and deploy to Github Pages

As I migrated my personal website from dreamhost to Github Pages, I need to change my wordpress blog into something that can live on Github Pages. I chose Hexo as Phil recommended to me several times.

It was a headache for me to set all things up, as neither I am a programmer nor I am familiar with github before. So here’s really handy post about how to set up a Hexo blog on your github pages as a project site.

Github Pages

Github Pages supports two different sites: user site and project site. I already have my portfolio set as my user site (with url the same as my github.io directory). I set this blog as a project site.

Here’s what I did:

  1. Create new Github repo
  2. Hexo setup & deploy
  3. Custom domain
  4. Write posts
  5. Customize blog style

Create new Github repo

  1. Create a new repository named ‘Blog’.
  2. Clone it to my local directory named .../blog/, using git clone dir

Remember Git and Github are different things! It’s crucial when using git and github. Here’s an article that can help.

Customize blog style

First, choose a theme that you like. Here’s a great resource made by Hexo: Hexo Themes.

  1. Choose a theme.
  2. Git clone the theme to your local themes folder under /blog/.
  3. Change theme setting in your _config.yml file.
