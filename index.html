Use Hexo and deploy to Github Pages

As I migrated my personal website from dreamhost to Github Pages, I need to change my wordpress blog into something that can live on Github Pages. I chose Hexo as Phil recommended to me several times.

It was a headache for me to set all things up, as neither I am a programmer nor I am familiar with github before. So here’s really handy post about how to set up a Hexo blog on your github pages as a project site.

Github Pages

Github Pages supports two different sites: user site and project site. I already have my portfolio set as my user site (with url the same as my github.io directory). I set this blog as a project site.

Here’s what I did:

  1. Create new Github repo
  2. Hexo setup & deploy
  3. Custom domain
  4. Write posts
  5. Customize blog style

Create new Github repo

  1. Create a new repository named ‘Blog’.
  2. Clone it to my local directory named ‘…/blog/‘, using git clone dir

Remember Git and Github are different things! It’s crucial when using git and github. Here’s an article that can help.

Hexo Setup

Before installing Hexo, you need to install:

  • Node.js
  • Git

As I already had both of these, I can move straight down to Hexo installation with npm.

Then go to my local dir ‘…/blog/‘, use:
$ hexo init …/blog/
$ cd …/blog
$ npm install

You will see there’re node_modles, public, source, themes folder and _config.yml, package.json under your blog folder. Then I will set the blog on your github pages first, by setting information in the _config.yml file under ‘…/blog/‘. Because I really want to see it live!

Here’s the documentation for the tags in config file.

You need to change the basic site information under #Site and the url under #URL. If your site is put in a subdirectory, set url as ‘http://yoursite.com/child‘ and root as ‘/child/‘. For example, mine would be:
url: http://zzhen.us/blog
root: /blog/
permalink: :year/:title/
permalink_defaults:

The #Deployment part is where you set your blog to be deployed on your github repo. If you are deploying your blog as user site, you will set the branch as master. If it is a project site as mine, you will set the branch as gh-pages. Mine would be:
deploy:
type: git
repo: git@github.com:withzzz/blog.git
branch: gh-pages

Hexo Deploy

After you setup the _config.yml file, your blog is mostly ready. Now, try:
$ hexo clean
$ hexo deploy
You don’t have to Launch automatic page generator in your github setting. Hexo will automatically publish your site on http://username.github.io/blog.

Custom domain

As I already set the CNAME file for customizing the domain of my github pages, I don’t need to customize again for my project site if I want it to to be the subsite of it.

Write Posts

You can use markdown language to write Hexo post. See Markdown Reference

Customize blog style

First, choose a theme that you like. Here’s a great resource made by Hexo: Hexo Themes.

  1. Choose a theme.
  2. Git clone the theme to your local themes folder under /blog/.
  3. Change theme setting in your _config.yml file.
